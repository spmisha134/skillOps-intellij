# Validation

Before finishing, validate the result using this checklist.

## Required checks

- [ ] The result follows the skill description.
- [ ] Repository instruction files were searched or considered.
- [ ] Existing project structure was inspected before suggesting structure.
- [ ] The solution follows existing naming and package conventions.
- [ ] The output is specific and actionable.
- [ ] The output avoids unnecessary architecture.
- [ ] The result includes testing or verification steps where relevant.
- [ ] Any assumptions are clearly stated.

## Failure conditions

The result is not acceptable if:

- it ignores existing project instructions
- it invents a new structure without checking the repository
- it gives generic advice without concrete next steps
- it changes unrelated files or behavior
- it makes unsupported assumptions
