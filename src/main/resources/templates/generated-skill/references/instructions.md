# Instructions

This file contains the detailed workflow for this skill.

## Goal

Use this skill to complete tasks that match the skill description in `SKILL.md`.

## Required workflow

1. Read the user request carefully.
2. Search for repository-level instruction files:
   - `AGENTS.md`
   - `agents.md`
   - `CLAUDE.md`
   - `claude.md`
   - `GEMINI.md`
   - `gemini.md`
3. Follow those project instructions before applying this skill.
4. Inspect the existing project structure before suggesting new structure.
5. Prefer adapting to the current project conventions.
6. Keep changes small, focused, and easy to review.
7. Do not invent frameworks, packages, or architecture that the project does not need.
8. When unsure, explain the assumption instead of silently guessing.

## Output guidance

The output should be practical and directly usable.

Prefer:

- clear steps
- concrete file paths
- specific class or function names
- validation/testing instructions
- small incremental implementation guidance

Avoid:

- vague recommendations
- unnecessary abstractions
- large rewrites
- unrelated improvements
