# Examples

## Good example

User request:

> Create a spec for adding validation to the skill generator.

Good response:

- Checks `AGENTS.md` or similar project instruction files first.
- Uses the existing package structure.
- Adds the spec under the existing docs/specs folder.
- Keeps the implementation scope focused.
- Defines acceptance criteria and test expectations.

## Bad example

User request:

> Create a spec for adding validation to the skill generator.

Bad response:

- Suggests a completely new architecture.
- Ignores existing docs.
- Creates unrelated packages.
- Does not mention tests.
- Does not explain how the result should be validated.
